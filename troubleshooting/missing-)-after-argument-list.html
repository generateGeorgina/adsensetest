Fix Windows error in three easy steps

When working with Windows computers, you may get an error shown below. For this, you need to use the path detailed in step 2, under "scripts": { etc. in the JSON file.

                    $ node --debug-brk ./node_modules/.bin/jest --runInBand
basedir=$(dirname "$(echo "$0" | sed -e 's,\\,/,g')")
          ^^^^^^^
SyntaxError: missing ) after argument list

1. Install Jest

Open Command Prompt on Windows, navigate to your current project directory. You can use the terminal in VS Code instead, in which case, you should be in the root (the place that contains all the relevant files/folders for your project) of your project already. Run the following command:

npm install --save-dev jest

You should see a new file, package.json in the root of your project directory.

2. Add the correct path for Windows to your JSON file

Add "type": "module" to your package.json file. This will be within the root curly braces (don't forget the comma). Do the same with "scripts": { "test": node ./node_modules/jest/bin/jest.js --runInBand"} so you can easily run npm run test:

                
{
    "type": "module",
    "scripts": {
            "test": "node ./node_modules/jest/bin/jest.js --runInBand"
        },
    
    "devDependencies": {
        "jest": "^29.7.0"
    }
}

3. Run npm run test

Since you have added the correct path to scripts and saved it under the name test in the previous step, you can now run npm run test.

npm run test
